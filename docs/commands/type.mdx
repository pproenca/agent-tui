## Usage

```bash
agent-tui type <text>
```

## Description

Types literal text characters into the active TUI session. Unlike `fill`, this sends individual keystrokes and doesn't target a specific element.

## Arguments

| Argument | Description |
|----------|-------------|
| `<text>` | Text to type |

## Examples

```bash
# Type a message
agent-tui type "Hello, World!"

# Type a command
agent-tui type "npm install express"

# Type into focused input
agent-tui type "my-project-name"
```

## Use Cases

### When to Use `type` vs `fill`

| Scenario | Command |
|----------|---------|
| Fill a detected input field | `fill @e1 "value"` |
| Type into focused element (no ref) | `type "value"` |
| Send text to command prompt | `type "command"` |
| Append to existing value | `type "more text"` |

### Example: Claude Code Interaction

```bash
# Spawn Claude Code
agent-tui spawn "claude --dangerously-skip-permissions"
agent-tui wait ">"

# Type a prompt (no element ref needed)
agent-tui type "Create a hello world script"
agent-tui keystroke Enter
```
