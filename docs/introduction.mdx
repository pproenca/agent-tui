## The Problem

Interactive CLI wizards block automation:

```bash
# This hangs waiting for user input!
npx create-next-app
# ? What is your project named? ...
```

AI agents can't "see" what's on screen or interact with TUI elements.

## The Solution

agent-tui captures screens, detects elements, and lets AI interact:

```bash
# 1. Spawn the wizard
agent-tui spawn "npx create-next-app"

# 2. See what's on screen
agent-tui snapshot -i
# @e1 [input] "project name" | @e2 [select] "TypeScript"...

# 3. Interact using element refs
agent-tui fill @e1 "my-app"
agent-tui keystroke Enter
```

## Why agent-tui?

| Problem | Solution |
|---------|----------|
| Interactive CLI wizards block automation | agent-tui captures screens and detects elements |
| No API for terminal UIs | Element refs (`@e1`, `@e2`) make TUIs scriptable |
| AI can't "see" terminal state | Snapshots provide structured view of screen + elements |
| TUIs require human interaction | AI agents can now fill forms, click buttons, navigate menus |

## Works With

agent-tui is designed for AI agents:

- **Claude Code** — Anthropic's AI coding assistant
- **Cursor** — AI-powered code editor
- **Codex** — OpenAI's code generation
- **Copilot** — GitHub's AI pair programmer
- **Any AI agent** — Any system that can run bash commands

  <Accordion icon="crosshairs" title="Element Detection">
